## Directory Purposes

**`src/`:**
- Purpose: React application source code
- Contains: JSX/JavaScript components, API client, styles
- Key files: `index.js` (app entry), `api/client.js` (data layer), `components/*.js` (UI)

**`src/components/`:**
- Purpose: Reusable React components for the UI
- Contains: Component definitions, local state management, styling
- Key files: `Sidebar.js`, `Grid.js`, `Inspector.js`

**`src/api/`:**
- Purpose: HTTP client layer abstracting WordPress REST API calls
- Contains: API fetch functions for folders, items, moves, tags, categories
- Key files: `client.js` (all API functions)

**`includes/`:**
- Purpose: PHP class definitions for WordPress plugin functionality
- Contains: Plugin initialization, REST routes, taxonomy setup, admin UI
- Key files: `class-wpmf-autoloader.php` (loader), `class-wpmf-api.php` (REST endpoints)

**`build/`:**
- Purpose: Compiled and minified assets ready for production
- Contains: Generated by `npm run build` using `@wordpress/scripts`
- Generated: Yes | Committed: No (typically in .gitignore)

**`tests/`:**
- Purpose: PHP unit tests for plugin functionality
- Contains: Test cases for API, taxonomy, tag mapper classes
- Key files: Test files follow naming pattern `test-*.php` or `*-test.php`

**`.planning/codebase/`:**
- Purpose: Generated GSD analysis documents
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md
- Generated: Yes | Committed: Yes

## Key File Locations

**Entry Points:**

**Backend Entry:**
- `wp-media-folders.php`: Main plugin file loaded by WordPress
  - Defines plugin constants (`WPMF_VERSION`, `WPMF_PLUGIN_DIR`, `WPMF_PLUGIN_URL`)
  - Requires autoloader and triggers initialization via `plugins_loaded` hook

**Frontend Entry:**
- `src/index.js`: React app root
  - Defines `App` component with state management
  - Sets up drag-and-drop context via `@dnd-kit`
  - Renders to `#wpmf-app-root` on DOM load

**Configuration:**

**Plugin Configuration:**
- `wp-media-folders.php`: Plugin header with name, description, version, author
- `includes/class-wpmf-*.php`: Individual feature configuration (taxonomy, API routes, meta fields)

**Build Configuration:**
- `package.json`: Build scripts, dependencies, version
  - `build`: Compile React via `wp-scripts build`
  - `start`: Watch mode for development
  - `package`: Create plugin zip via `wp-scripts plugin-zip`
  - `test:unit`: Run JavaScript tests via `wp-scripts test-unit-js`
  - `test:php`: Run PHP tests via wp-env

**WordPress Testing:**
- `.wp-env.json`: Local WordPress environment configuration for development/testing

**Core Logic:**

**React Components:**
- `src/components/Sidebar.js`: Folder navigation tree with creation UI
- `src/components/Grid.js`: Media items display with pagination
- `src/components/Inspector.js`: Folder metadata editor (tag mapping)

**API Client:**
- `src/api/client.js`: All REST API calls
  - `getFolders()`: Fetch folder taxonomy
  - `createFolder()`: POST new folder
  - `getItems()`: Fetch media items with pagination
  - `moveItems()`: Move items to folder

**REST Endpoints:**
- `includes/class-wpmf-api.php`: Custom REST route registration
  - `POST /wpmf/v1/move`: Move items to folder with permission checks

**Taxonomy:**
- `includes/class-wpmf-taxonomy.php`: Register `wp_virtual_folder` custom taxonomy
  - Hierarchical, mapped to attachment and product post types
  - REST enabled for API access

**Tag Mapping:**
- `includes/class-wpmf-tag-mapper.php`: Register term metadata fields
  - `wpmf_mapped_post_tags`: Array of WP tag IDs bound to folder
  - `wpmf_mapped_product_cats`: Array of Woo category IDs bound to folder

**Testing:**

**Frontend Tests:**
- `src/components/Sidebar.test.js`: Unit tests for Sidebar component
- Run via: `npm run test:unit`

**PHP Tests:**
- `tests/` directory: PHP test files (location convention, not yet populated)
- Run via: `npm run test:php`

## Naming Conventions

**Files:**

**PHP Classes:**
- Pattern: `class-{plugin-prefix}-{feature-name}.php`
- Example: `class-wpmf-api.php`, `class-wpmf-taxonomy.php`
- Convention: Class name matches file name in PascalCase (WPMF_API, WPMF_Taxonomy)

**JavaScript/React Components:**
- Pattern: `{ComponentName}.js` (PascalCase)
- Example: `Sidebar.js`, `Grid.js`, `Inspector.js`, `MediaItem` (component within Grid)
- Convention: Default export is the main component

**Test Files:**
- Pattern: `{ComponentName}.test.js` (JavaScript), `test-{feature-name}.php` (PHP)
- Example: `Sidebar.test.js`

**Styles:**
- Pattern: `style.scss` (global) or co-located with components via imports
- Example: `src/style.scss`

**Directories:**

- **`src/`** - Source code root
- **`src/api/`** - API client layer
- **`src/components/`** - React components
- **`includes/`** - PHP business logic
- **`build/`** - Compiled output
- **`tests/`** - Test suites

## Where to Add New Code

**New Feature (e.g., bulk operations):**
- Primary code: `src/components/NewFeature.js` (React component)
- API support: Add function to `src/api/client.js`
- Backend: Add class to `includes/class-wpmf-{feature}.php`
- REST endpoint: Add route in `includes/class-wpmf-api.php`
- Tests: `src/components/NewFeature.test.js` and `tests/test-{feature}.php`

**New Component/Module (e.g., modal dialog):**
- Implementation: `src/components/{ComponentName}.js`
- Import in parent component where needed
- Styles: Inline via `style` prop or import from `src/style.scss`
- Tests: `src/components/{ComponentName}.test.js`

**New API Endpoint (e.g., batch delete):**
- Route registration: `includes/class-wpmf-api.php::register_routes()`
- Handler method: `includes/class-wpmf-api.php::handle_operation()`
- Client function: `src/api/client.js::operationName()`
- Component call: Use new client function in component `handleOperation()`

**Utilities/Helpers:**
- Shared frontend utilities: `src/utils/` (directory doesn't exist; create as needed)
- Shared PHP utilities: `includes/class-wpmf-helpers.php` (create as needed)
- Example: `buildTree()` in Sidebar.js is a local utility; move to `src/utils/tree.js` if reused

**Styles:**
- Global styles: `src/style.scss`
- Component-specific: Inline via `style` prop or import from global

**Hooks (future React features):**
- Custom hooks: `src/hooks/useHookName.js` (directory doesn't exist; create as needed)
- Example: `useFolders()` for folder fetching/caching
