In `src/components/Inspector.js`, in the `handleSave` function, replace the unconditional `setTermCache(updatedCache)` call (currently line 102) with a conditional call that only fires when new terms were actually created.

Current code (lines 88-103):
```javascript
const results = await Promise.allSettled([...tagCreations, ...catCreations]);
const updatedCache = { ...termCache };
results.forEach((result, index) => {
    if (result.status === 'fulfilled') {
        const term = result.value;
        const type = index < tagCreations.length ? 'tag' : 'category';
        updatedCache[term.name] = { id: term.id, type };
    } else {
        console.error('Failed to create term:', result.reason);
    }
});

// Update termCache with newly created terms before ID lookup
setTermCache(updatedCache);
```

Replace the last two lines (the comment and `setTermCache(updatedCache)`) with:

```javascript
// Only update termCache when new entries were added — avoids
// re-triggering useEffect([folderId, termCache]) which clears saveStatus.
const newEntries = results.filter(r => r.status === 'fulfilled');
if (newEntries.length > 0) {
    setTermCache(updatedCache);
}
```

The rest of handleSave continues to use `updatedCache` (not the state variable) for the tagIds/catIds lookup, so this change does not affect ID resolution -- `updatedCache` is a local variable that already has the new entries regardless of whether state was updated.

Do NOT change the useEffect dependency array `[folderId, termCache]` -- that is correct. The fix is to avoid triggering a termCache state change when no new terms were created.
