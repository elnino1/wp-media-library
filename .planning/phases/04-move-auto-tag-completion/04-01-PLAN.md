3. Add a new public static method `apply_tags_for_folder` with this exact signature and body:

```php
public static function apply_tags_for_folder( int $item_id, int $folder_id ): void {
    $tag_ids = get_term_meta( $folder_id, 'wpmf_mapped_post_tags', true );
    if ( ! is_array( $tag_ids ) || empty( $tag_ids ) ) {
        return;
    }
    // append=false replaces existing tags (MOVE-02 requires replacement, not append)
    wp_set_object_terms( $item_id, $tag_ids, 'post_tag', false );
}
```

In `includes/class-wpmf-admin-page.php`, in the `enqueue_scripts` method:

The current code passes `$assets['dependencies']` directly to `wp_enqueue_script`. The `build/index.asset.php` file is auto-generated by `@wordpress/scripts` from JS imports, and it does NOT include `wp-media` because Grid.js accesses `window.wp.media` directly (not via an import).

After the line `$assets = require $asset_file;` (currently line 35), add:

```php
$deps = array_merge( $assets['dependencies'], array( 'media-views' ) );
```

Note: The correct WordPress script handle for the media uploader is `media-views`, not `wp-media`. The `media-views` handle loads `wp.media` into the global scope. This is the standard WP handle.

Then change the `wp_enqueue_script` call to use `$deps` instead of `$assets['dependencies']`:

```php
wp_enqueue_script(
    'wpmf-app-js',
    WPMF_PLUGIN_URL . 'build/index.js',
    $deps,
    $assets['version'],
    true
);
```
