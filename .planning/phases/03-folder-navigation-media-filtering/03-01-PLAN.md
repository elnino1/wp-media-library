Create `register_rest_fields()` method that hooks into the REST media query:
```php
public static function register_rest_fields() {
    add_filter( 'rest_attachment_query', array( __CLASS__, 'filter_media_by_folder' ), 10, 2 );

```

Create `filter_media_by_folder( $args, $request )` method:
- Check if the request has a `wpmf_folder` parameter: `$request->get_param('wpmf_folder')`
- If `wpmf_folder` param is present and equals `'inbox'` (string), add a tax_query to get media with NO `wp_virtual_folder` terms:
  ```php
  $args['tax_query'] = array(
      array(
          'taxonomy' => 'wp_virtual_folder',
          'operator' => 'NOT EXISTS',
      ),
  );
  ```
- If `wpmf_folder` is a numeric value (a folder ID), add a tax_query to get media assigned to that specific term:
  ```php
  $args['tax_query'] = array(
      array(
          'taxonomy' => 'wp_virtual_folder',
          'field'    => 'term_id',
          'terms'    => array( absint( $wpmf_folder ) ),
      ),
  );
  ```
- If no `wpmf_folder` param, return `$args` unmodified (backward compatible).
- Always return `$args`.

This approach uses WordPress's built-in `rest_attachment_query` filter rather than creating a custom endpoint, keeping the architecture simple.

**In `src/api/client.js` -- update `getItems` function:**

Replace the current `getItems` implementation. The new version must:
1. Accept `folderId`, `page`, `perPage` params (same signature)
2. Build queryArgs with `per_page` and `page` (same as now)
3. Add folder filtering:
   - If `folderId` is `null`, `undefined`, or `0`: add `wpmf_folder: 'inbox'` to queryArgs (inbox = unassigned media)
   - If `folderId` is a positive number: add `wpmf_folder: folderId` to queryArgs
4. Keep the same `/wp/v2/media` endpoint with `addQueryArgs`
5. Keep the same try/catch error handling returning `[]` on failure

The key change is adding `wpmf_folder` to the queryArgs object before calling `addQueryArgs`. Remove the comment about "future WooCommerce support" since the function now does real filtering.

```javascript
export const getItems = async (folderId = null, page = 1, perPage = 20) => {
    const queryArgs = {
        per_page: perPage,
        page: page,
        wpmf_folder: folderId ? folderId : 'inbox',
    };

    const path = addQueryArgs('/wp/v2/media', queryArgs);

    try {
        const response = await apiFetch({ path });
        return response || [];
    } catch (error) {
        console.error('Error fetching items:', error);
        return [];
    }
};
```
